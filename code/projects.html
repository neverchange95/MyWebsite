MP3-Player in Java

Meine erste praktische Erfahrung einen Sourcecode eigenständig zu entwickeln wurde mir im Modul Grundlagen der Programmierung 2 an meiner Hochschule vermittelt. Zum erfolgreichen bestehen des Moduls musste neben der Prüfung eine praktische Aufgabe erledigt werden, welche darin bestand einen MP3-Player mithilfe von einigen Tipps in der Programmiersprache Java zu entwerfen. Die Gesamtaufgabe wurde in einzelne Teilaufgaben zerlegt, welche jeweils binnen einer Woche abgegeben werden mussten. Der jeweils implementierte Code konnte mit JUnit Tests auf die erforderliche Funktionalität vor der Abgabe getestet werden. Das Programm ermöglicht es dem Benutzer eine Playlist mit seinen gewünschten Songs zu erstellen und diese mithilfe des Players abzuspielen.

Rootkit & Keylogger

Im Rahmen einer Seminararbeit im Bereich IT-Security entwickelte ich ein Kernel-Mode-Rootkit für den Linux Kernel sowie einen Keylogger, welcher das Schadprogamm darstellen sollte. Das Rootkit wurde für den Linux Kernel 4.10.0-38-generic entwickelt und hat die Aufgabe bestimmte Prozess Identifier vor dem Linux Programm "ps", welches alle aktuell laufenden Prozesse auflistet zu verbergen. Um dies zu erreichen wurde die Adresse der Funktion, welche die einzelnen Verzeichniseinträge (also die laufenden Prozesse) an das Programm "ps" liefert in der System-Call-Table durch die Adresse des Rootkits ersetzt. Das Rootkit übernimmt dann im Prinzip die gleiche Aufgabe wie die ursprüngliche Funktion, gibt aber den oder die Verzeichniseinträge, die verborgen werden sollen nicht mit zurück. Dadurch wird erreicht, dass dem Benutzer einzelne laufende Prozesse nicht mehr angezeigt werden und sie somit unsichtbar sind. Der zusätzlich entwickelte Keylogger diente als eigentliches Schadprogramm, welcher mithilfe des Rootkits dann vor dem Anwender verborgen wurde. Die Aufgabe des Keyloggers war es, wie der Name schon sagt, alle vom Benutzer eingetippten Tastatureingaben in eine Datei mitzuschreiben.

Visuelle Tiefen- und Breitensuche

Eine kleine Visualisierung der bekannten Tiefen- und Breitensuche, wobei die beiden Algorithmen in einem Labirinth von einem Startpunkt aus ein bestimmtes Ziel suchen. Das Labirinth wird in einer canvas mithilfe von kleinen Quadraten dargestellt. Jedes Quadrat repränentiert dabei ein Stück Wand oder ein Stück vom Weg. Werden die Suchalgorithmen gestartet, wird der Weg, den sie bereits gegangen sind solange schwarz eingefärbt, bis sie das Ziel gefunden haben. Beide Algorithmen können gleichzeitig gestartet werden, um die jeweiligen Unterschiede in Echtzeit zu sehen.
